# Jersey BESY — веб-админка

Панель на **React** + **API** (Node) с ролями **администратор** и **тренер**, синхронизация с мобильным приложением (**Flutter**) по секретному заголовку.

## Запуск

```bash
cd dance_school_admin
npm install
npm run dev
```

- API: **http://localhost:5050**
- Сайт: **http://localhost:3000** (откроется экран **входа**)

## Роли

### Администратор

- Расписание (шаблоны занятий)
- Все записи из приложения
- Тренеры
- Отчёты (в т.ч. записи по тренерам)

**Тестовый вход:** логин `admin`, пароль `admin123`.

При первом запуске, если в `data.json` нет учётных записей (массив `accounts` пустой или отсутствует), сервер **сам создаёт** администратора и аккаунты тренеров (см. ниже).

### Тренер

- Только **свои** записи (где `trainerName` совпадает с профилем)
- **Чаты** только со своим потоком (`dialogs[имя тренера]`)
- Ответы ученикам через форму внизу чата

## API и безопасность

| Метод | Путь | Кто |
|--------|------|-----|
| POST | `/api/auth/login` | Все (логин/пароль) |
| GET | `/api/studio` | JWT админа (полные данные), JWT тренера (срез), **или** заголовок `X-Sync-Token` (полные данные для Flutter) |
| PUT | `/api/studio` | JWT **админа** или `X-Sync-Token` (мобильное приложение) |
| POST | `/api/chat/message` | Мобильное приложение (`X-Sync-Token`) |
| POST | `/api/trainer/message` | JWT **тренера** |

Переменные окружения (опционально):

- `JWT_SECRET` — секрет подписи JWT (смените в проде)
- `MOBILE_SYNC_TOKEN` — токен для Flutter (по умолчанию `dev-sync-token`)
- `DATABASE_URL` — строка подключения PostgreSQL (если задана, чаты пишутся в БД)

Пример `DATABASE_URL`:

```bash
postgres://postgres:password@localhost:5432/dance_school
```

При включенной PostgreSQL сервер сам создаёт таблицу `chat_messages`, сообщения в чате сохраняются туда.

## Связь с Flutter

Приложение шлёт заголовок **`X-Sync-Token`** (в коде Flutter: `STUDIO_SYNC_TOKEN`, по умолчанию совпадает с сервером).

Пример для телефона в Wi‑Fi:

```bash
flutter run ^
  --dart-define=STUDIO_API_URL=http://192.168.1.10:5050 ^
  --dart-define=ADMIN_WEB_URL=http://192.168.1.10:3000 ^
  --dart-define=STUDIO_SYNC_TOKEN=dev-sync-token
```

Если на сервере сменили `MOBILE_SYNC_TOKEN`, передайте то же значение в `STUDIO_SYNC_TOKEN`.

## Формат `server/data.json`

- `accounts` — учётные записи (пароли только в виде **bcrypt** `passwordHash`)
- `trainers`, `classTemplates`, `bookings`, `dialogs` — как раньше
